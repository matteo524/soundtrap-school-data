Section 1

Overview & Architecture

This is a single-file static HTML prototype (trial-form.html) with no backend dependencies. It demonstrates how school and district data hosted on GitHub Pages can power a cascading school lookup in a trial request form.

⚠️ Prototype only. Form submission currently shows a success banner but does not send data anywhere. All backend integration (STWeb, BigQuery, Salesforce notifications) must be built separately before going to production.

Tech stack

  • Vanilla JavaScript (ES5-compatible — no build tools, no frameworks)
  • Single HTML file with embedded CSS and JS
  • School data served as static JSON from GitHub Pages
  • No cookies, no tracking, no third-party dependencies except Google Fonts
  • localStorage used only for the "Welcome Back" pre-fill banner

Section 2

GitHub-Hosted School Data

All school data is served as static JSON files from GitHub Pages at:

https://matteo524.github.io/soundtrap-school-data/

index.json

Fetched once on page load. Contains the full US school hierarchy:

  • List of US states
  • Per state: list of cities
  • Per city: list of district names
  • List of available per-state school files (slugs)

This file powers the Country → State → City → District cascade instantly, without additional network requests.

schools-{state-slug}.json

One file per US state (e.g. schools-california.json). Fetched silently in the background as soon as the user selects a state. Contains:

  • Every school in that state, keyed by district name (lowercase)
  • Each school entry: { name: "School Name", nces: "123456789" }
  • A special _enrollment key with district-level student population figures (from NCES 2022–23 data)
Data coverage: US public schools only, sourced from NCES. Private schools and non-US schools are not in the database. The data is read-only — only GitHub collaborators with write access can modify the JSON files.

Performance strategy

  • index.json is fetched on page init so the state/city/district cascade is instant
  • The per-state schools-*.json file starts downloading in the background the moment a state is selected — by the time the user picks a district, the data is usually already available
  • A client-side in-memory cache prevents duplicate fetches within the same page session

Section 5

Fallback Flows — "Can't find" paths

When a school or district is not in the database, the user can fall back to manual text entry. In all fallback cases, no NCES ID is captured.

"I can't find my district" Public school only

Available only in public school mode for US users. Opens a manual entry panel where the user types all details free-form:

  • Country (pre-filled, locked to United States)
  • State, City, District name, School name (all free text)
  • School website (optional — helps Salesforce validate later)

No JSON is queried. Submitting this path triggers the Salesforce validation workflow.

"I can't find my school" US · Public + Private

Behaviour differs slightly by school type:

Public schoolPrivate school
Requires district?Yes — must be selected from cascade firstNo
Pre-filled fieldsCountry, State, City, District (all locked)State, City (locked); no district field
User typesSchool name onlySchool name only

In both cases, submitting triggers the Salesforce validation workflow.

No NCES ID for non-US schools. All non-US submissions will have no NCES ID and will always route through the Salesforce validation workflow. The Salesforce team will need to create a unique identifier for each non-US school.

What happens after a non-US submission

  1. Trial is activated immediately
  2. Salesforce team is notified (school not in database)
  3. Team validates whether the school already exists in any internal system
  4. If found: link existing identifier to the STWeb account
  5. If not found: create a new unique identifier → add to STWeb → add school entry to the GitHub JSON (for future lookups if the school is in the US) or maintain in Salesforce/CRM for non-US

Section 8

School Identifiers & Key Design Decisions

NCES ID as primary key

The NCES ID (National Center for Education Statistics) is the primary identifier for US public schools. It is a unique 12-digit number assigned by the US Department of Education. When present, it is the definitive link between:

  • The GitHub JSON school database
  • The STWeb account
  • The BigQuery trial record

Non-US / manual entry schools

For schools without an NCES ID, a unique internal identifier must be created by the Salesforce team. This should follow a consistent naming convention to be defined (e.g. ST-{COUNTRY}-{ID}). This identifier replaces the NCES ID in STWeb and should be added to the GitHub JSON if the school is later confirmed as a US institution.

Switching school type resets all fields

By design, switching between Public and Private school type performs a full reset of all cascade fields (country, state, city, district, school, NCES, enrollment). This prevents stale or locked state from carrying over between modes.

Section 10

Other Dev Notes

Welcome Back banner (localStorage)

If a user has previously submitted the form, their contact info (name, email, subject, school) is saved to localStorage. On their next visit, a "Welcome Back" banner appears offering to pre-fill the form. No server-side session is involved — this is entirely client-side and clears if the user clears browser storage.

CORS

GitHub Pages serves files with permissive CORS headers (Access-Control-Allow-Origin: *), so the JSON files can be fetched from any domain — including a future HubSpot landing page or custom domain.

Combobox implementation

All dropdowns (country, state, city, district, school) use a custom combobox implementation with live text search filtering. This is important for the school field in particular, which can contain hundreds of entries. The underlying <select> elements are hidden and kept in sync — they are the authoritative source of truth for form values.

Offline / slow connection behaviour

If index.json fails to load, the state/city/district cascade will not function. If a per-state schools-*.json fails to load, the school dropdown will show "No schools found" and the user is expected to use the "I can't find my school" fallback. There are no retry mechanisms in the prototype.

Browser compatibility

Written in ES5-compatible JavaScript. Works in all modern browsers. Does not require any polyfills.

Deployment

Upload trial-form.html to the root of the soundtrap-school-data GitHub repo. GitHub Pages will serve it at:

https://matteo524.github.io/soundtrap-school-data/trial-form.html

No build step, no CI, no environment variables needed.
